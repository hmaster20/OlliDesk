# Управление зависимостями пакетов

Вы используете стандартную структуру метаданных PEP 621 ([project]) со сборщиком Setuptools ([tool.setuptools]).

Классический pip не умеет автоматически модифицировать такой файл при установке пакетов.
Чтобы не переписывать `pyproject.toml` вручную, лучшим решением для вашего стека будет использование утилиты uv от Astral. Она разработана специально для того, чтобы мгновенно работать с вашим текущим файлом напрямую, без переключения на тяжеловесный Poetry.

Вот как автоматически добавлять пакеты в ваш конфигурационный файл:

## Решение: Использование uv (Рекомендуется)

Утилита uv полностью понимает секции [project.dependencies] и [project.optional-dependencies] из вашего файла и автоматически прописывает туда новые пакеты при установке.

1. Установите uv в систему (один раз, через PowerShell от администратора):

```shell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh | iex"

# или в локальное окружение

pip install uv
```

2. Как устанавливать пакеты теперь:

```sh
# Чтобы добавить обычную зависимость в dependencies:
uv add bump-my-version

# Чтобы добавить пакет в вашу dev-группу [project.optional-dependencies] (так как bump-my-version — это инструмент разработчика):
uv add --optional dev bump-my-version

# чтобы избежать ошибки Failed to hardlink files, добавляем --link-mode=copy
uv add --optional dev bump-my-version --link-mode=copy

uv add --dev types-PyYAML --link-mode=copy
```

Что произойдет автоматически:

* uv сам найдет ваш .venv (или создаст новый, если нужно).
* Скачает и установит пакет в виртуальное окружение (в 10-100 раз быстрее, чем pip).
* Автоматически добавит строку "bump-my-version>=1.4.1" внутрь вашего pyproject.toml в соответствующую секцию.

------------------------------

## Как это выглядит прямо сейчас в вашем pyproject.toml

Если вы примените команду uv add --optional dev bump-my-version, утилита автоматически приведет вашу секцию dev к следующему виду:

[project.optional-dependencies]
dev = [
    "pytest>=8.0",
    "pytest-asyncio>=0.23",
    "pytest-mock>=3.12",
    "pytest-cov>=4.1",
    "ruff>=0.3",
    "mypy>=1.9",
    "black>=24.0",
    "isort>=5.13",
    "bump-my-version>=1.4.1", # <- Строка добавится сама
]

Если вы пока не хотите ставить сторонние утилиты, вам придется вносить строчку "bump-my-version>=1.4.1" в массив dev вручную через текстовый редактор, а затем выполнять привычный pip install -e .[dev].
Какой подход вам ближе — попробовать uv для автоматизации или настроить генерацию через другие инструменты?

------------------------------

Чтобы обновить зависимости и при этом ничего не сломать, нельзя обновлять всё вслепую. Ваше опасение абсолютно оправданно: мажорные обновления (например, переход с Fastapi 0.x на 1.x или Pydantic 1 на 2) гарантированно сломают код.
Вот безопасный алгоритм действий с использованием uv.

------------------------------
## Шаг 1. Безопасное обновление (Только патчи и минорные версии)
Самый безопасный способ — обновить пакеты в рамках текущих версий, прописанных в вашем pyproject.toml. Это обновит баг-фиксы, но не затронет глобально логику.
Выполните команду:

uv pip compile pyproject.toml --upgrade --output-file uv.lock

(Или, если вы используете новый формат проектов uv):

uv lock --upgrade

Что это сделает:

* Перезапишет ваш lock-файл.
* Обновит пакеты до максимально возможных, не нарушая ограничений в pyproject.toml (например, если у вас написано fastapi>=0.130.0,<0.140.0, она никогда не поставит 0.141.0).

------------------------------
## Шаг 2. Как посмотреть, что изменится ДО установки?
Чтобы не гадать, какие пакеты затронуты, запустите синхронизацию в режиме dry-run (симуляция):

uv sync --dry-run

(Или для старого интерфейса: uv pip sync uv.lock --dry-run)
Вы увидите точный список со стрелочками вроде → fastapi==0.139.0. Если вы заметите там критически важный для вас пакет, обновление которого пугает, вы сможете его зафиксировать.
------------------------------
## Шаг 3. Как защитить конкретные пакеты от обновлений?
Если вы боитесь обновлять условный torch или fastapi, жестко зафиксируйте их версии прямо в файле pyproject.toml в секции dependencies:

dependencies = [
    "fastapi==0.138.1",       # Будет устанавливаться ТОЛЬКО эта версия
    "torch>=2.12.0,<2.13.0",   # Разрешены только безопасные патчи (2.12.1, 2.12.2)
    "numpy~=2.5.0",           # Эквивалентно >=2.5.0, <2.6.0
]

------------------------------

## Золотое правило безопасности

   1. Используйте Git: Перед любой командой обновления убедитесь, что у вас чистый git status.
   2. Если после uv sync что-то пошло не так, вы просто делаете git checkout -- uv.lock (или pyproject.toml) и возвращаете всё назад одной командой.
