# Руководство пользователя OlliDesk

## Что такое OlliDesk?

OlliDesk — это десктопный интерфейс в стиле IDE для локальных AI-ассистентов кода, работающих через **Ollama**. Он предоставляет панель чата с контекстом проекта, встроенный браузер файлов и веб-редактор на основе Monaco — всё на базе локальной LLM для генерации, объяснения и рефакторинга кода.

## Быстрый старт

1. **Откройте проект:** Запустите OlliDesk и выберите директорию проекта через **File → Open Project**.
2. **Выберите модель:** Убедитесь, что в Ollama загружена модель (например `qwen2.5-coder:7b`), и выберите её в выпадающем списке на панели статуса.
3. **Начинайте кодить:** Используйте панель чата справа или редактор слева для навигации и изменения файлов.
4. **Задавайте вопросы:** Пишите вопросы или инструкции в поле ввода чата внизу справа.

## Схема главного окна

```
+------------------+----------------------------+
|  Дерево файлов   |  Редактор / Чат           |
|  (левая панель)  |  (основная область)       |
|                  |                            |
|  - src/          |  [Вкладка редактора]      |
|  - tests/        |                            |
|  - README.md     |  +-------------------------+
|                  |  | Панель чата (справа)   |
|                  |  |                         |
|                  |  | В: "Добавить обработку" |
|                  |  | О: [ответ модели]      |
+------------------+----------------------------+
|  Статус-бар: [модель] | [путь к проекту]     |
+------------------+----------------------------+
```

- **Дерево файлов** — просмотр и выбор файлов проекта. Двойной клик — открыть в редакторе.
- **Редактор** — полнофункциональный редактор кода (Monaco, тот же движок, что в VS Code). Поддержка вкладок.
- **Панель чата** — общение с LLM. Поддерживает **Markdown** в ответах.
- **Статус-бар** — показывает текущую модель, путь к проекту и переключатель темы.

## Возможности

### Управление проектами

- **Открыть проект** — откройте любую директорию как проект. Дерево файлов обновится автоматически.
- **Недавние проекты** — быстрый доступ через меню File.
- **Состояние проекта** — OlliDesk запоминает открытые файлы, вкладки и сессию для каждого проекта.

### Редактирование кода

- **Monaco Editor** — подсветка синтаксиса, IntelliSense, мультикурсор, поиск и замена.
- **RAG-контекст** — векторное хранилище автоматически индексирует файлы проекта для интеллектуального поиска контекста во время чата.
- **Обзор файлов** — навигация по дереву проекта, открытие нескольких файлов во вкладках.

### Чат с LLM

- **Контекстная осведомлённость** — ассистент видит текущий активный файл и релевантный контекст проекта.
- **Markdown-рендеринг** — ответы отображаются с подсвеченными блоками кода.
- **Отображение "размышлений"** — если модель отправляет `reasoning_content`, он показывается в сворачиваемом блоке.
- **Остановка генерации** — нажмите кнопку остановки, чтобы прервать длинный ответ.
- **Авто-прокрутка** — новый контент появляется автоматически.

### Сессии

- **Сохранение сессий** — история переписки сохраняется для каждого проекта и восстанавливается при повторном открытии.
- **Новая сессия** — начните новый разговор через **Session → New Session**.
- **Управление сессиями** — переключайтесь между несколькими сессиями в рамках одного проекта.

### Тема

- **Внешний вид** — переключение между **светлой** и **тёмной** темой через **View → Toggle Theme** или кнопку на панели статуса.
- **Сохранение** — выбранная тема запоминается.

### Помощь с кодом

- **Объяснение кода** — вставьте или укажите функцию и попросите объяснить её.
- **Генерация кода** — опишите, что нужно, и модель напишет код.
- **Рефакторинг** — попросите улучшить код, переименовать или оптимизировать.
- **Помощь с отладкой** — поделитесь сообщением об ошибке или проблемным кодом и получите предложения.

## Конфигурация

OlliDesk хранит настройки в `~/.ollidesk/`:

| Файл | Назначение |
|------|------------|
| `config.yaml` | Глобальные настройки (модель, тема и т.д.) |
| `sessions.db` | База данных истории сессий |
| `vector_db/` | Векторное хранилище ChromaDB для RAG |
| `logs/` | Логи приложения |

### Мастер первого запуска

При первом запуске мастер настройки запросит:

- **Модель по умолчанию** — модель Ollama для чата.
- **Эмбеддинг-модель** — модель для индексации RAG (по умолчанию: `nomic-embed-text`).

### Ручная настройка

Вы можете отредактировать `~/.ollidesk/config.yaml` напрямую:

```yaml
default_model: qwen2.5-coder:7b
embed_model: nomic-embed-text
theme: dark
wizard_dismissed: true
```

## Требования

- **Ollama** — установлен и запущен (с хотя бы одной загруженной моделью).
- Python 3.10+ (при запуске из исходного кода).
- Windows 10+ (или Linux/macOS с соответствующей сборкой).

## Решение проблем

| Проблема | Решение |
|----------|---------|
| "Connection refused" при запуске | Убедитесь, что Ollama запущена (`ollama serve`) |
| Модель не отвечает | Загрузите модель: `ollama pull qwen2.5-coder:7b` |
| Редактор Monaco пустой | Проверьте, что `ui/web_editor/vendor/monaco/` содержит файлы редактора |
| Дерево файлов пустое | Используйте **File → Open Project** для выбора директории |
| История чата исчезла | Сессии хранятся в `~/.ollidesk/sessions.db` |

## Часто задаваемые вопросы

**В: Можно ли использовать любую модель Ollama?**
О: Да, любую модель из вашего локального Ollama. Кодо-специфичные модели (qwen2.5-coder, codellama, deepseek-coder) работают лучше всего.

**В: Отправляет ли OlliDesk данные куда-либо?**
О: Нет. Все вычисления выполняются локально на вашем компьютере через Ollama. Никакие данные не покидают ваше устройство.

**В: Можно ли использовать OlliDesk без GPU?**
О: Да, но большие модели будут работать медленнее на CPU. Небольшие модели вроде `qwen2.5-coder:1.5b` хорошо работают на CPU.
